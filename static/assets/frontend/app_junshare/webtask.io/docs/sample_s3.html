Restrict Access to Data in S3

Custom authorization policy for accessing data in S3

For customized documentation and ready to run samples, please log in.

In this sample you will create a webtask token that allows uploading or downloading data from your S3 bucket. The webtask code behind this token can implement arbitrary authorization policies that restrict access to the S3 bucket in ways that cannot be expressed with the declarative policies supported by AWS. The webtask token encrypts your AWS credentials, which means you can safely embed the token in your mobile or SPA application without the risk of disclosing your AWS keys.

While the example below uses S3, the same method can be applied to provide custom authorization logic to any web services that are protected with a simple key credential you don't want to embed directly in your mobile or SPA application, e.g. Twilio, Azure Blob Storage, Stripe, and more.

The webtask token that we are going to create will allow callers to perform upload and download of objects with names matching a pre-configured regular expression from a pre-configured S3 bucket in a given region. This is the data you will need to run the sample; we are going to assume it is available in environment variables as follows:

export AWS_REGION={AWS region name, e.g. us-west-1}
export S3_BUCKET={S3 bucket name the must already exists in the region}
export ACCESS_KEY_ID={Your AWS access key ID}
export SECRET_ACCESS_KEY={Your AWS secret access key}

For demonstration purposes, the authorization policy applied by the webtask code will be based on restricting the S3 object names that can be downloaded or uploaded using regular expressions. Let's define two regular expressions: one for uploading and another for downloading resources:

export PUT_REGEX=^foo
export GET_REGEX=^foo

Regular expressions above will allow both upload and download of objects with names beginning with foo from the S3_BUCKET. Feel free to come up with a different regular expression.

We are now going to issue a new webtask token that:

  • Specifies the webtask code to execute via URL pointing directly to GitHub.
  • Includes the S3_BUCKET, AWS_REGION, PUT_REGEX, and GET_REGEX as plaintext, signed parameters. This means the owner of the webtask token can see them but cannot modify them.
  • Inludes the ACCESS_KEY_ID and SECRET_ACCESS_KEY as encrypted and signed parameters. This means the owner of the webtask token cannot obtain their value or modify them. The parameters will only be provided in plaintext form to the webtask code (which you control) at runtime.
  • Restricts the webtask container name in which the webtask will run to

First let's store the payload of the request we will issue to create an S3 webtask token in the s3_token_request.json file:

cat > s3_token_request.json <<EOF
{
  "url": "https://raw.githubusercontent.com/auth0/webtask-scripts/master/s3/access_s3.js",
  "ten": "{webtask_container1}",
  "pctx": {
  "region": "$AWS_REGION",
  "bucket": "$S3_BUCKET",
  "put_regex": "$PUT_REGEX",
  "get_regex": "$GET_REGEX"
  },
    "ectx": {
    "access_key_id": "$ACCESS_KEY_ID",
    "secret_access_key": "$SECRET_ACCESS_KEY"
  }
}
EOF

Now, let's use curl to make a webtask token request and obtain the S3_TOKEN which allows restricted access to S3:

export S3_TOKEN=$(curl -s https://webtask.it.auth0.com/api/tokens/issue -H "Authorization: Bearer {webtask_token}" -H "Content-Type: application/json" --data-binary @s3_token_request.json)

You can now use the newly issued token to store data in S3:

curl https://webtask.it.auth0.com/api/run/{webtask_container1}?path=foo_1.txt\&method=put -H "Authorization: Bearer $S3_TOKEN" --data-binary 'Hello, world!'

You can also use the token to get data from S3:

curl https://webtask.it.auth0.com/{webtask_container1}?path=foo_1.txt\&method=get -H "Authorization: Bearer $S3_TOKEN" -X POST

Note how an attempt to upload data to an S3 object with a name that does not match the pre-defined regular expression is rejected:

curl https://webtask.it.auth0.com/api/run/{webtask_container1}?path=restricted.txt\&method=put -H "Authorization: Bearer $S3_TOKEN" --data-binary 'Hello, world!' -v

NOTE the S3_TOKEN webtask token is safe to embed in a mobile or SPA application without the risk of disclosing your AWS credentials. The owner of the token can only upload or download data from your S3 bucket subject to the S3 object name restrictions expressed in the regular expressions.
